# NSAP Scheme Eligibility Prediction

Multi-class classification model that predicts the most appropriate NSAP
pension scheme:
- `IGNOAPS` — Old Age Pension
- `IGNWPS` — Widow Pension
- `IGNDPS` — Disability Pension

This project includes a Flask web API and model training pipeline.

## GitHub-ready Contents
Upload these files to GitHub:
- `app.py` — Flask API and web interface
- `train_model.py` — training script to create model artifacts
- `requirements.txt` — Python package dependencies
- `Dockerfile` — container build instructions
- `README.md` — project documentation
- `.gitignore` — files to exclude from version control

> The following files are excluded from GitHub uploads using `.gitignore`:
> `nsap_model.pkl`, `nsap_scaler.pkl`, `nsap_label_encoder.pkl`, `nsap_features.pkl`, and the dataset CSV.

## Local Setup
Install dependencies and run the app locally:
```bash
pip install -r requirements.txt
python app.py
```
Then open `http://localhost:8080` in your browser.

To retrain or regenerate the model artifacts, place the dataset CSV in the project
folder and run:
```bash
python train_model.py
```

## API Example
Call the API directly with JSON input:
```bash
curl -X POST http://localhost:8080/predict \
  -H "Content-Type: application/json" \
  -d '{"male_pct":0.0,"female_pct":1.0,"transgender_pct":0.0,"sc_pct":0.1,"st_pct":0.05,"gen_pct":0.7,"obc_pct":0.15,"aadhaar_coverage_pct":0.95,"mobile_coverage_pct":0.8,"beneficiary_density":0.2}'
```

## Notes
- `.gitignore` is configured so that trained model files and the dataset are not uploaded accidentally.
- If you want to share the project without large binaries, keep only code + docs in GitHub.
- The model artifacts can be regenerated by running `train_model.py` when the dataset is available.
